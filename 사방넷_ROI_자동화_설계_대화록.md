## 📋 전체 설계 플랜 (Claude Code 작성본)

### 핵심 개념 — 어떤 코드를 기준으로 해야 하나?

**❌ 채널상품코드를 그대로 쓰면 안 되는 이유**

| 채널 | 상품코드 예시 | 문제 |
|------|-------------|------|
| ESM옥션 | GV001122334455 (14자리) | 앞 10자리만 매칭해야 함 |
| 스마트스토어 | NSS_00012345 | 완전 다른 형식 |
| 카카오선물하기 | KKO_GIFT_9876 | 또 다른 형식 |

채널코드 기준으로 집계하면 채널이 늘어날수록 예외 처리가 폭발적으로 증가함.

**✅ 정답: 사방넷 물류코드(연결상품코드)를 ROI 마스터 키로 사용**

| 비유 | 설명 |
|------|------|
| 각 채널 코드 | 각 나라 화폐 (달러, 엔, 유로) |
| **물류코드** | **원화 (공통 기준)** |
| 전채널 판매코드 | 환전소 (번역 테이블) |

물류코드가 좋은 이유:
- 내가 직접 부여하는 코드 → 채널이 바뀌어도 변하지 않음
- 원가표(전 제품 원가표)와 직접 연결됨

## 🔍 단계별 확인 Q&A

### Q1. 사방넷 주문수집 엑셀에 있는 상품 식별 코드

| 코드 | 있음/없음 | 비고 |
|------|---------|------|
| 채널상품코드 | ✅ | |
| 옵션코드 | ✅ (신뢰 불가) | 사방넷 등록 옵션 코드와 맞지 않음 |
| 판매코드 | ✅ | |
| 물류코드 | ✅ | |
| 쇼핑몰명 | ✅ | |
| 매출 | ✅ | |
| 주문수량 | ✅ | |

**확인 필요 항목:** 채널별 판매코드 → 물류코드 매핑 (세트 포함)

> 사방넷 **상품관리 → 판매상품 관리 → 전체 조회 → 엑셀 다운로드** 에서 확인 가능

---

### Q2. 세트 상품 구조

- 세트 + 단품 모두 존재
- 세트 구성 수에 따라 **1줄~5줄 내외**로 rawdata에 표시
- **핵심 이슈:** 물류코드 3개 매핑된 세트도 주문 수량은 `1`로 기재 → 원가 계산 오류 발생

---

### Q3. 원가 관리 방식

- **별도 원가표** (사방넷 외부 엑셀)로 관리
- 사방넷 물류코드에 자체상품코드(SKU)가 등록되어 있음
- 흐름: `물류코드 → SKU → 원가표 → 단가`

---

### Q4. 세트 구성 정보 위치

- **사방넷에 등록되어 있음** (연결상품 구성)

---

### Q5. 채널 ↔ 판매코드 관계

- 판매코드 1개 → 여러 채널 연결 **(동일 상품, 채널만 다름)**
- 판매코드 1개 → 채널 1개 **(특정 채널 전용)**
- → 채널 구분은 `쇼핑몰명`으로, 상품 구분은 `판매코드`로

---

### Q6. 세트 주문 여러 줄일 때 매출 위치

**케이스 A: 세트 3구성 → 1줄**
```
판매코드: SET-001 | 물류코드: WL-100 | 매출: 30,000원
```

**케이스 B: 세트 3구성 → 3줄 (첫 번째 줄에만 매출)**
```
1행: SET-001 | WL-100 | 매출: 30,000원  ← 여기만 매출
2행: SET-001 | WL-200 | 매출: 0
3행: SET-001 | WL-300 | 매출: 0
```

→ 매출은 1행에만, **원가는 3행 모두 합산**해야 ROI 정확

---

### Q7. 같은 세트 주문 행 묶는 키

- **주문번호** + **송장번호** 로 같은 주문 묶기 가능

---

### Q8. 수수료율

아래 케이스 모두 존재:
- 상시 vs 행사 때 수수료 다름 (예: GS shop 상시 22% / 행사 17%)
- 상품 카테고리에 따라 수수료 다름
- 시즌/프로모션에 따라 수시로 변경

**처리 방법:**
1. `채널명 + 주문날짜` 기준으로 판단
2. 노션 행사 일정 체크 → 행사 기간이면 행사 수수료율 적용
3. 아니면 기본 수수료율 적용

---

### Q9. 광고비

| 채널 유형 | 배분 방식 |
|----------|---------|
| 일부 채널 | 주차별 해당 채널 내 균일 분배 |
| 일부 채널 | 월별 균일 분배 |
| 일부 채널 | 건당 CPC |

- 광고비 총액은 **아직 한 곳에 정리 안 됨 → 새로 입력 양식 설계 필요**
- CPC 채널은 **따로 받아서 수동 입력**
- 란MD가 채널별 주차 or 월별 광고비 수동 제공 → 기준에 따라 자동 분배

---

### Q10. 배송비 계산 로직

브랜드별·채널별 배송비 기준이 달라 아래 수식 형태로 적용:

```excel
=IF(OR(D2="",COUNTIF($D$2:D2,D2)>1), 0,
  IF(E2="스마트스토어(ncp_1ntfei_01)",
    IF(J2<50000, -900, 2600),
    IF(AND(K2=0, H2="비피젠", E2<>"스마트스토어(ncp_1ntfei_01)"), 2250,
      IF(AND(K2=0, H2<>"비피젠", E2<>"스마트스토어(ncp_1ntfei_01)"), 2600,
        IF(AND(K2=4000, H2="비피젠", E2<>"스마트스토어(ncp_1ntfei_01)"), -1750,
          IF(AND(K2=3500, H2<>"비피젠", E2<>"스마트스토어(ncp_1ntfei_01)"), -900,
          0))))))
```

**배송비 계산에 필요한 변수:**

| 변수 | 역할 |
|------|------|
| 송장번호 | 중복 행 제거 (세트 2번째 행은 0) |
| 채널명 | 스마트스토어 별도 로직 |
| 매출 | 스마트스토어 50,000원 기준 |
| rawdata 배송비 수집값 | 0 / 3,500 / 4,000 등 |
| 브랜드명 | 비피젠 vs 일반 구분 |

### Q11. 원가표 구조

- SKU 코드 + **브랜드명** 함께 있음
- 물류코드 컬럼 없음 → SKU만 있음
- 물류코드 ↔ SKU 매핑은 별도로 관리

## 📊 오류 없는 매칭 흐름 (3단계)

```
사방넷 rawdata (주문 1건)
├─ 채널상품코드
└─ 옵션 바코드

     ↓ 1단계: 채널상품코드 완전일치 매칭
     
전채널_판매코드 → 물류코드, 최종원가, 수수료율

     ↓ 실패 시 2단계: ESM 채널이면 앞 10자리로 재매칭
     ↓ 실패 시 3단계: 바코드로 매칭

     ↓ 매칭 성공
     
공헌이익 = 매출 - 배송비(계산) - 원가 - 수수료(수수료율×매출) - 광고비
```

| 시트 | 내용 |

| **⑤ 시각화 시트** | 표·차트로 시각적으로 확인 |

## 🔧 특수 케이스 처리 방법

| 케이스 | 문제 | 처리법 |
|--------|------|--------|
| 세트 분리 주문 | 같은 주문번호·송장번호가 여러 줄 | 2번째 행부터 매출·배송비 = 0 |

## ✅ 검증 방법

1. 수동으로 한 주치 rawdata 붙여넣었을 때의 공헌이익 합계 계산
2. 자동화 실행 후 동일한 값이 나오는지 비교
3. 미매칭 리스트(마스터 데이터에 없는 코드)가 0이 되도록 마스터 데이터 보완
